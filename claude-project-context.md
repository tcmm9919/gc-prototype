# 🏛 GC Project — Freedom AI Compliance Platform

## Цель документа
Продуктовый контекст и информационная архитектура **Freedom AI Compliance Platform (GCP)** — внутренней AML/KYC/комплаенс-платформы Freedom Bank Kazakhstan. Прототип для демонстрации UX заказчику до начала dev-фазы.

---

## 1. Что это за продукт

GCP — внутренняя платформа для офицеров комплаенса банка, которая:
- мониторит транзакции и клиентов в реальном времени
- срабатывает по AML/KYC-правилам и сценариям
- автоматизирует обработку алертов через AI-агента
- ведёт расследования (кейсы) с документами и эскалациями
- логирует всё в аудит-журнал
- управляет ML-моделями для скоринга и детекции аномалий

**Локаль:** Казахстан, валюта KZT, таймзона Asia/Almaty, язык русский.

**Эталонная боевая среда (от которой редизайним):** `box-mock.ffb-dev.freedombank.kz`

---

## 2. Пользователи и их роли

### Live users (люди)
- **Compliance Officer** — обрабатывает алерты, ведёт кейсы, делает EDD — **основная роль**
- **Risk Manager** — настраивает риск-факторы, правила, сценарии
- **Data Analyst** — работает с ML-моделями, метриками, фичестором
- **Administrator** — управление учётками, ролями, системными настройками
- **Auditor** — read-only доступ к журналу

### System actors
- **Compliance Officer AI** — автономный LLM-агент, формально пользователь системы. Один singleton. Обрабатывает ~80% алертов автоматически без участия человека.

## 3. Jobs To Be Done (JTBD)

### Compliance Officer (главная роль)

| Когда | Хочу | Чтобы |
|---|---|---|
| Захожу утром на работу | Увидеть что произошло за ночь и что мне нужно сделать | Не пропустить критическое + расставить приоритеты |
| Прилетел новый алерт | Понять контекст: правило, клиент, транзакция | Решить — false positive, эскалировать, или создать кейс |
| Веду расследование | Собрать все артефакты (документы, EDD, доказательства) в одном месте | Принять обоснованное решение |
| Знаю что нужна рутинная проверка клиента | Запустить уже существующий workflow одним кликом | Не делать руками то, что делает AI |
| Проверяю конкретного клиента | Видеть полный профиль рисков, историю операций, скоры от агентов | Быстро оценить уровень доверия |

### Compliance Officer AI (системный)

| Триггер | Действие |
|---|---|
| Новый alert | Получаю → анализирую (правило + транзакция + клиент) → решаю: запустить workflow, эскалировать, или пропустить |
| Workflow выполнен клиентом | Перепроверяю правила по обновлённым данным → закрываю кейс автоматически если правило больше не срабатывает |
| Решение неоднозначное | Эскалирую менеджеру письмом |

### Risk Manager
- Настроить пороги риск-факторов → влияют на скоринг
- Создать новое правило (атомарное условие) → используется в сценариях
- Создать сценарий (комбинация правил + триггеры + действия) → автоматизация

### Auditor
- Просмотреть журнал действий по любой сущности
- Фильтр по пользователю / типу действия / IP / временному периоду
- Экспорт CSV для compliance-отчётности

## 4. Доменные сущности

| Сущность | Что это | ID-формат |
|---|---|---|
| **Клиент** (Client) | Физ. или юр. лицо на мониторинге. Имеет ИИН/БИН, скоринг, риск-уровень, флаги (PEP, sanctions, EDD), теги | `CL-S01`, `CL-0001`, `CL-KILLER-DYAGILEV` |
| **Транзакция** (Transaction) | Операция клиента с реквизитами, кодом назначения, риск-флагами, привязкой к сценариям | `TX-S001`, UUID для боевых |
| **Правило** (Rule) | Атомарное условие над сущностью (клиент/транзакция/группа). Поле + оператор + значение. Используется в сценариях | `RL-XXX` |
| **Оповещение** (Alert) | Событие срабатывания правила. Severity: low/medium/high/critical. Может связываться с кейсом | `AL-S01`, UUID |
| **Кейс** (Case) | Расследование — объединяет 1+ алертов, транзакций, клиентов. Имеет тип, статус, исполнителя, SLA, доказательства | `CASE-YYYYMMDD-XXXXXXXX` (sequential hex suffix) |
| **Сценарий** (Workflow) | Drag-and-drop pipeline из 15 типизированных активностей. Запускается на одного клиента (или группу). Может вызывать ML-модели и WARM | `SC-XXX`, `#001l` для боевых |
| **EDD** | Расширенная проверка клиента — 10-секционный отчёт от EDD-агента + чек-лист 6 шагов | Часть Client |
| **WARM** | Длительный (часы-дни) бизнес-процесс комплексной проверки клиента. Состояние pipeline видно отдельно | Status view |
| **AI Agent** | Пользовательский LLM-агент с промптом, моделью, инструментами. Подключается в сценариях | `AG-1`, `AG-2`, `AG-3` |
| **Compliance Officer AI** | Мета-агент-singleton (см. §6) | Не имеет ID, это сама система |
| **ML-модель** | Knowledge base без UI-операций: Bank Offline Feature Store, TSAD, CTSM | `bank-offline-fs`, `tsad`, `ctsm` |
| **Риск-фактор** | Динамический атрибут с весом, влияющий на скоринг | `RF-XXX` |

## 5. Информационная архитектура (все экраны)

### МОНИТОРИНГ

**`/dashboard`** — Главная
- Баннер «Compliance Officer AI закрыл N кейсов автоматически и сэкономил X часов»
- 6 KPI: Клиенты / Открытые оповещения / Активные кейсы / Транзакции / Активные правила / Критические
- Activity stream (один row = один case, статус-summary, фильтр по статусу)
- Distribution sidebar: Оповещения (по важности) + Кейсы (по статусу) — отдельные карточки

**`/clients`** — Реестр клиентов
- Таблица: ФИО / Тип (физ/юр) / Риск / Скоринг / Статус / Ответственный / Последняя операция
- Фильтры: Риск / Статус / Тип / поиск по имени-ID

**`/clients/[id]`** — Карточка клиента (9 табов)
- Обзор: ПРОФИЛЬ + КАНАЛ УВЕДОМЛЕНИЙ + ПОДРОБНОСТИ (10 полей) + ФЛАГИ (15 флагов) + AI-ГЕНЕРАЦИЯ СЦЕНАРИЕВ + ИЗБРАННЫЕ СЦЕНАРИИ + ДОКУМЕНТЫ
- Скоринг: Профиль рисков с подскорами от 4 агентов (Аномальные транзакции / Внутренний скоринг / Новостной агент / EDD-агент)
- Транзакции / Оповещения / Кейсы — таблицы по клиенту
- EDD: чек-лист 6 шагов + 10-секционный отчёт + кнопка Скачать PDF
- Новости: adverse media с тональностью POSITIVE/NEGATIVE
- История: log изменений (стикер 5)
- Документы: загруженные паспорта, доходы, юр. документы

**`/transactions`** — Реестр операций
- Колонки: Дата / ID / Клиент / Код назначения / Сумма / В тенге / Комплаенс / Приоритет / Риск / Филиал

**`/transactions/[id]`** — Карточка транзакции
- PUBLIC ID + клиент + комплаенс-статус + AI Ассистент
- 4 секции: Информация о транзакции / Клиент / Назначение / Участники
- Банковская информация (счета, филиал, ордер)

**`/chat`** — Общий AI-чат

- Стрим ответа, цитирование сущностей (CL-XXX, TX-XXX и т.д.)
- Выбор модели

**`/rules`** — Правила (атомарные условия)

## 6. Compliance Officer AI — главный нарратив продукта

Системный singleton-агент, который автономно обрабатывает алерты. **Сердце ценностного предложения.**

### Принцип работы (6 шагов)
1. Получает каждый новый алерт
2. Анализирует контекст: правило + транзакция + клиент
3. Принимает решение:
   - `dispatch_workflow` — создаёт авто-кейс и запускает подходящий workflow
   - `escalate` — эскалирует менеджеру письмом
   - `pass` — false positive, закрывает
4. Если запустил workflow — ожидает completion от клиента (до 7 дней)
5. Перепроверяет правила по обновлённым данным
6. Если правило больше не срабатывает → закрывает кейс автоматически. Иначе → эскалирует.

### Где виден в UI
- В журнале аудита — как пользователь `Compliance Officer AI`
- В кейсах — как `Исполнитель: Compliance Officer AI`
- В LLM Usage — как агент `compliance_agent`
- На `/agents/compliance-officer` — настройки + activity log
- На dashboard — отдельный stream активности

### Setting (на странице агента)
- Email менеджера для отчётов
- 4 чекбокса «Когда отправлять письма» (при dispatch / при clear / при fail / при escalate)
- Кнопка Активен / Выключить

## 7. Killer Flow — демо-история

Преднастроенный end-to-end сценарий по клиенту **Дягилев Михаил Владимирович**, прослеживается на 6+ экранах одновременно. Активация: DevToolbar → «Открыть killer flow».

```
Транзакция 1.5M KZT (код 119, для физ. лица с нулевым balance baseline)
  ↓ сработало правило «Транзакционная аномалия»
Alert (severity high)
  ↓ Compliance Officer AI принимает решение
Решение LLM: dispatch_workflow → «Income Proof Remediation»
  ↓ агент создаёт авто-кейс
Case CASE-20260506-7F6A3D24 (исполнитель = Compliance Officer AI)
  ↓ workflow запускается на клиента
Workflow: 6 шагов (email клиенту → upload PDF → OCR → extract income → recalc baseline → update KYC)
  ↓ клиент загружает income_anomaly_remediation.pdf
OCR извлекает: «Средняя выплата 6 240 000 ₸/мес»
  ↓ обновляется declared_monthly_income клиента
Rules re-check: правило больше не срабатывает
  ↓ агент закрывает кейс
Case status: Решено + письмо менеджеру «закрыт автоматически»
```

**Где «оседает» killer flow:**
- `/clients/CL-KILLER-DYAGILEV` — клиент с обновлённым declared_monthly_income, алерт в табе Оповещения, кейс в табе Кейсы, scoring с подскорами от 4 агентов, EDD-отчёт
- `/cases/CASE-20260506-7F6A3D24` — авто-кейс с исполнителем AI, комментариями, доказательствами, журналом
- `/workflows/SC-KILLER-REMEDIATION` — preconfigured workflow
- `/audit` — 7 audit events (создание alert → создание case → run workflow → recalc risk → update case → отправка report)
- `/llm-usage` — 10 LLM-вызовов (compliance_agent + converter), ~$0.20 за решение
- `/alerts/AL-KILLER-01` — alert со статусом «rejected» (т.к. был автоматически разобран)

## 9. Тематика и эстетика

### Текущее направление: **Revolut Dark**
- **Body:** глубокий тёмный с лёгким зелёным undertone, едва заметный radial glow в верхне-левом углу, правая часть почти чёрная
- **Sidebar:** прозрачный, 88px vertical tiles (icon top + label bottom), белый текст, fill-style иконки
- **Header:** прозрачный, большой page-title слева (28px Bold), справа кластер с search / bell / theme / Freedom Bank org-pill / avatar
- **Cards:** alpha-white (8% over body), без borders, без shadows
- **KPI:** числа большим mono, дельта-pill с TrendingUp/Down
- **Status pills:** rounded-full с tinted bg + цветной dot

### Дизайн-принципы (см. design.md)
- Nielsen heuristics, Gestalt principles, UX laws (Jakob's, Hick's, Fitts's, Miller's)
- 4 обязательных состояния каждого экрана: Empty / Loading / Data / Error
- Один primary CTA на экран
- Microcopy = глагол + объект
- Motion 200-300ms, ease-out для входа

### Альтернативные направления (рассмотрены, отложены)
- **Pro Terminal** — Linear/Vercel-style dark с monospace акцентами
- **Editorial Premium** — Fraunces serif на кремовом, magazine-style
- **Sovereign Banking** — Mercury-style off-white с ambient glow
